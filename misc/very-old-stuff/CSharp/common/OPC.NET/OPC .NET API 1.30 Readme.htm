OPC .NET API 1.30 Readme

November 23, 2004

Overview

The latest version of this document always on the OPC foundation website.

Any use of the OPC .NET API SDK or OPC .NET API Source Code is subject to the terms the OPC Foundation License Agreement.

The OPC .NET API is primarily intended to provide interoperability between existing OPC specifications and applications developed in the .NET environment. The API does support COM based and SOAP/XML based servers via the same set of interfaces, however, the API does not introduce new concepts or data models. In other words, the .NET API is intended provide an intermediate step between the existing OPC technologies and the new technologies which are part of the Unified Architecture initiative.

Distribution

The OPC .NET API is distributed in two ways:  

File

Description
OPC .NET API 1.30 Merge Module Installs the redistributable binaries.
OPC .NET API 1.30 SDK Installs the binaries and help documentation.
OPC .NET API 1.30 Source Code Installs the binaries, runtime configuration files and all source.

The Source Code distribution includes a shareware control library found at http://www.netcontrols.org/scpl/. This library is only used by the HDA 1.20 Sample Client and is not part of the SDK distribution.

System Requirements

Operating Systems

The OPC .NET API requires that the .NET Framework Redistributable 1.1 or the .NET Framework SDK Version 1.1 is present on the target system. The .NET framework does not support Windows 95. 

All source code provided by the OPC Foundation is only supported on Windows 2000 SP3 and Windows XP SP1. That said; the source code has been designed to be portable to other operating systems. For this reason vendors are encouraged to report fixes required to port the samples to other operating systems so they can be incorporated into the baseline and to improve the portability of future sample code.

Installers

The installer packages are self-installing files; however, they require that Windows Installer 2.0 be present on the system (it is standard on Windows 2000 SP3 and Windows XP). The Windows Installer 2.0 support files for other operating systems can be downloaded from Microsoft's website at:

Download Windows Installer 2.0 Redistributable for Windows NT 4.0 and 2000  

Download Windows Installer 2.0 Redistributable for Windows 95, 98 and ME

Source Code

The source code projects require either Visual Studio 6.0 or Visual Studio .NET 2003. 

Visual Studio .NET 2003 can be installed along side both Visual Studio .NET 2002 and Visual Studio 6.0. Existing projects for older development environments do not need to be upgraded in order to use sample code distributed by the OPC Foundation.

Usage

SDK

You may add the OPC .NET API to your VS .NET projects by opening the 'Add Reference' dialog and navigating to $(CommonProgramFiles). Once this is done the VS .NET intellisense should automatically pick up the documentation files.

The OpcRegSvr utility operates like regsvr32. For example the following command registers the sample COM server:

OpcRegSvr /r .\OpcDaSampleServer.dll

Source Code

The source code builds the assemblies and signs them with a test key-pair. This means that any application that uses the .NET API must be changed so that it references the new assemblies 

Problem Reporting

As part of a broader initiative to increase adoption of OPC specifications and to improve service to members, the OPC Foundation intends to maintain binaries and sample code over time following a model similar to Open Source projects. Vendors are encouraged to report problems, bug fixes, enhancements and any other comments about the components supplied by the OPC Foundation. To support this effort, the OPC Foundation has technical staff prepared to review and incorporate feedback into the baseline and release new versions of the components and sample code in a timely manner for the benefit of all members.

Please note that this support is intended for OPC Foundation members who are incorporating OPC support into their products.

Problems related to a particular download should be posted to the appropriate group on the OPC Foundation Message Board.
